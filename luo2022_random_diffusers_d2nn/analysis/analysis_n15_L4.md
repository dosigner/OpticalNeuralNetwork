# 실험 분석: n15_L4

> [!info] 실험 요약
> **모델**: 4-레이어 위상 전용 D2NN (240×240 그리드)
> **디퓨저 수**: epoch당 15개 (높은 다양성)
> **학습**: 30 epochs, B=64 (유효 배치 B×n=960), A100 TF32
> **최종 PCC**: ==0.879== | **최종 Loss**: -0.953 | **소요 시간**: 79.5분

---

## 1. 모델 구성

| 파라미터 | 값 |
|---------|-----|
| 레이어 수 | 4 |
| 그리드 | 240 × 240, pitch 0.3mm |
| 파장 | 0.75mm (400 GHz THz) |
| 전파 방식 | BL-ASM (Band-Limited Angular Spectrum) |
| 디퓨저 → L1 | 2mm |
| 레이어 간 | 2mm |
| L4 → 출력면 | 7mm |
| 위상 초기화 | Uniform [0, 2π] |
| 학습 파라미터 수 | 230,400 (4 × 57,600) |

> [!note] 디퓨저 모델
> - 얇은 랜덤 위상 디퓨저 ($\Delta n = 0.74$)
> - 상관 길이 ~$10\lambda$, 높이 표준편차 $8\lambda$
> - epoch마다 15개의 새로운 디퓨저를 생성하여 학습

## 2. 학습 동역학

### 수렴 곡선 분석

| 구간 | Epoch | PCC | 관찰 |
|------|-------|-----|------|
| 급속 상승 | 1→5 | 0.621 → 0.856 | 기본 산란 보정 학습 |
| 안정화 | 5→10 | 0.856 → 0.874 | 디퓨저 다양성에 적응 |
| 고원기 | 10→20 | 0.874 → 0.878 | 미세 조정 단계 |
| 최종 | 20→30 | 0.878 → 0.879 | 수렴 완료 |

> [!important] 핵심 관찰
> n=15는 n=10 (PCC 0.884)과 거의 동일한 최종 성능을 보이지만, **수렴이 더 느림** (epoch당 160초 vs 108초). 이는 epoch당 처리해야 할 디퓨저-객체 쌍이 50% 더 많기 때문 (B×n = 960 vs 640).

### n=1, n=10과의 비교

| 모델 | 최종 PCC (train) | 소요 시간 | epoch당 시간 |
|------|-----------------|----------|-------------|
| n1_L4 | **0.907** | 7.6분 | ~15초 |
| n10_L4 | **0.884** | 53.8분 | ~108초 |
| n15_L4 | **0.879** | 79.5분 | ~160초 |

## 3. 논문 (Luo et al., eLight 2022)과의 비교

### 예상 결과와의 일치

논문의 핵심 주장:

1. **디퓨저 수 증가 → 학습 PCC 감소**: n=15의 학습 PCC (0.879)가 n=1 (0.907)보다 낮은 것은 예상대로. 더 많은 디퓨저를 동시에 만족시켜야 하므로 최적화가 더 어려움.

2. **디퓨저 수 증가 → 일반화 성능 향상**: 학습 PCC는 낮지만, **새로운 (blind) 디퓨저에 대한 PCC는 크게 향상**되는 것이 논문의 핵심 발견. n=15는 n=10보다 더 많은 디퓨저 다양성으로 학습하므로, 이전에 보지 못한 디퓨저에 대한 일반화가 더 우수할 것으로 예상.

3. **수확 체감 (Diminishing Returns)**: n=10에서 n=15로 증가 시 학습 PCC 차이 (0.884 → 0.879)는 n=1에서 n=10 (0.907 → 0.884) 대비 크게 줄어듦. 이는 논문에서 보고한 $n \geq 10$ 이후의 수확 체감 패턴과 일치.

> [!tip] 논문 대비 해석
> 학습 PCC만으로는 모델의 실제 성능을 판단할 수 없음. **blind-test PCC** (fig5, fig6에서 평가)가 진정한 성능 지표. n=15는 학습 PCC가 n=1보다 낮지만, 실제 사용 시나리오 (임의의 새 디퓨저)에서는 더 높은 PCC를 보일 것으로 예상.

## 4. 물리적 해석

### 왜 디퓨저 다양성이 중요한가?

$$t_d(\mathbf{r}) = e^{j\phi_d(\mathbf{r})}$$

- **n=1**: 네트워크가 특정 $\phi_d(\mathbf{r})$의 역변환을 암기. 다른 디퓨저에는 무력.
- **n=15**: 네트워크가 다양한 $\phi_d$ 패턴에 공통적으로 작동하는 **산란 불변 특징**을 학습. 특정 디퓨저의 세부 구조가 아닌, 산란의 통계적 특성에 적응.

### 학습 메커니즘

각 epoch에서 15개 디퓨저에 대해 동일한 위상 레이어 $\phi_m$을 최적화:

$$\mathcal{L} = -\frac{1}{15B}\sum_{i=1}^{15}\sum_{j=1}^{B} \text{PCC}(I_{ij}^{\text{out}}, I_j^{\text{target}}) + \alpha \cdot \mathcal{E}_{\text{penalty}}$$

이 과정은 디퓨저 다양성이 **정규화 효과**를 가짐:
- 과적합 방지 (특정 디퓨저 암기 불가)
- 일반적인 산란 보정 전략 학습 유도

## 5. Fig 3 (n-sweep) 재현에 대한 시사점

n15_L4는 n-sweep 연구의 세 번째 데이터포인트:

```
n=1  → PCC_train=0.907, PCC_blind=? (낮을 것으로 예상)
n=10 → PCC_train=0.884, PCC_blind=? (중간)
n=15 → PCC_train=0.879, PCC_blind=? (높을 것으로 예상)
n=20 → (학습 중)
```

> [!warning] n=20 학습 대기 중
> fig3 완성을 위해 n20_L4 학습 완료 필요. 현재 GPU에서 진행 중.

## 6. 계산 비용 분석

| 항목 | n1_L4 | n10_L4 | n15_L4 | 배율 |
|------|-------|--------|--------|------|
| epoch당 시간 | 15초 | 108초 | 160초 | ×10.7 |
| 전체 학습 시간 | 7.6분 | 53.8분 | 79.5분 | ×10.5 |
| 유효 배치 크기 | 64 | 640 | 960 | ×15 |
| GPU 메모리 | ~2GB | ~14GB | ~21GB | ×10.5 |

> [!note] 메모리 스케일링
> B×n=960에서 A100 40GB의 약 53%를 사용. n=20 (B×n=1280)은 약 70%로 추정되며, A100 한계 내에서 처리 가능.

## 7. 결론 및 다음 단계

1. n=15는 논문의 예상대로 **학습 PCC가 n=10보다 약간 낮음** (0.879 vs 0.884)
2. 수확 체감 패턴이 관찰됨 — $n=10 \to 15$에서의 PCC 감소폭이 $n=1 \to 10$보다 작음
3. **blind-test 평가** (fig5, fig6)에서 n=15의 일반화 우위가 확인될 것으로 예상
4. n=20 학습 완료 후 전체 n-sweep (fig3) 재현 가능
